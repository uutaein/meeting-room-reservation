## 1. Context

반복 예약은 매주 같은 요일의 예약 회차를 한 번에 만든다.

그런데 회의실은 업무일 기준으로 사용하는 자원이며, 공휴일에는 회의실 예약을 하지 않는 것으로 정책을 확정한다.

따라서 반복 예약 기간 안에 공휴일이 포함되면, 해당 공휴일 회차를 그대로 생성하면 안 된다.

---

## 2. Decision

> 반복 예약 미리보기에서 공휴일 회차를 별도 표시한다.
>
> 반복 예약 생성 시에는 공휴일 회차를 제외한다.

세부 결정은 다음과 같다.

1. 반복 예약 날짜 계산은 반복 종료월 마지막 날까지 수행한다.
2. 계산된 회차 중 공휴일인 날짜는 `excludedHolidays`로 분리한다.
3. 미리보기 화면은 생성 예정 날짜, 충돌 날짜, 공휴일 제외 날짜를 구분해서 보여준다.
4. 실제 생성 API는 공휴일 회차를 생성하지 않는다.
5. 공휴일 회차 제외는 사용자가 별도 선택하지 않아도 기본 적용한다.

---

## 3. Rationale

1. 공휴일에는 회의실 예약을 하지 않는다는 운영 기준과 맞다.
2. 반복 예약 생성 후 공휴일 회차를 사람이 직접 취소하는 일을 줄인다.
3. 미리보기에서 제외 날짜를 보여주면 사용자가 생성 결과를 예측할 수 있다.
4. 단건 예약의 공휴일 불가 정책과 반복 예약 정책이 일관된다.

---

## 4. Consequences

### Positive

1. 공휴일 예약이 자동 생성되지 않는다.
2. 반복 예약 결과가 업무일 기준에 가까워진다.
3. 사용자는 공휴일 때문에 빠진 회차를 미리 알 수 있다.

### Risk

1. 공휴일 데이터가 오래되면 제외 판단이 틀릴 수 있다.
2. 임시공휴일은 데이터 갱신 전까지 반영되지 않을 수 있다.

### Mitigation

1. 공휴일 데이터 지원 연도를 명시한다.
2. 지원하지 않는 연도는 오류로 처리한다.
3. 임시공휴일은 수동 데이터 갱신 절차로 관리한다.
