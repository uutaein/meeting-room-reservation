## 1. 정책 변경

반복 예약의 공휴일 처리 정책을 다음과 같이 확정한다.

> 반복 예약 미리보기에서 공휴일 회차를 별도 표시한다.
>
> 반복 예약 생성 시에는 공휴일 회차를 제외한다.

---

## 2. 사용자 가치

1. 사용자는 반복 예약 생성 전 실제 생성될 날짜를 정확히 알 수 있다.
2. 사용자는 공휴일 때문에 빠지는 회차를 미리 확인할 수 있다.
3. 공휴일 예약이 자동으로 생성되는 일을 막을 수 있다.
4. 생성 후 공휴일 예약을 사람이 직접 취소하는 일을 줄일 수 있다.

---

## 3. 화면 방향

반복 예약 미리보기에는 다음 목록을 구분해서 표시한다.

| 목록 | 설명 |
|---|---|
| 생성 예정 날짜 | 실제 예약 생성 대상 날짜 |
| 충돌 날짜 | 기존 ACTIVE 예약과 시간이 겹치는 날짜 |
| 공휴일 제외 날짜 | 공휴일이라 예약 생성에서 제외되는 날짜 |

공휴일 제외 날짜는 경고가 아니라 안내 정보로 표시한다.

---

## 4. 생성 정책

1. 시스템은 반복 종료월 마지막 날까지 매주 같은 요일의 회차를 계산한다.
2. 계산된 회차 중 공휴일은 생성 대상에서 제외한다.
3. 계산된 회차 중 기존 예약과 충돌하는 날짜는 충돌 날짜로 분리한다.
4. 실제 생성은 공휴일도 아니고 충돌도 아닌 날짜만 대상으로 한다.
5. 공휴일 제외는 사용자가 별도 선택하지 않아도 기본 적용한다.

---

## 5. 성공 기준

1. 반복 예약 미리보기에서 공휴일 제외 날짜가 표시된다.
2. 반복 예약 생성 결과에 공휴일 회차가 포함되지 않는다.
3. 생성 예정 건수는 공휴일 제외 후 실제 생성 가능한 건수를 기준으로 표시한다.
4. 전체 계산 대상 건수, 충돌 건수, 공휴일 제외 건수, 생성 가능 건수를 사용자가 구분할 수 있다.
